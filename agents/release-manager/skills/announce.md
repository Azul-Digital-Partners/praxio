# /announce — Release Communication Draft

**Trigger:** `/announce [release notes or branch]`

**Output:** Short internal announcement for Rosalind/CEO; optional external-facing version for Marketing

**Steps:**
1. Read the release notes (from /release output or provided)
2. Write internal version: 3–5 bullets, what shipped, what it enables, what's next
3. Flag anything that needs CEO awareness (breaking changes, scope shifts, blockers cleared)
4. If user-facing: pass to Marketing for external copy — do not write external comms yourself
