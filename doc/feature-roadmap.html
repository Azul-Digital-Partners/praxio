Conversation Interface

The core Praxio paradigm — talk to your agent team instead of managing a task board
✦ Praxio
Today — Daily Command Center
Phase 2
Always-live view pinned to the top of the left nav. Rosalind is always present. You open Praxio and just talk — drop an update, capture a task, mark something done, ask a question, make a decision. No routing required on your end. Rosalind reads the stream, creates tasks, updates state files, dispatches to other agents, and responds when you need a response.

Each calendar day is its own Today thread. At midnight it closes and becomes a permanent daily log entry. The next morning a fresh Today opens with Rosalind already loaded with yesterday's summary and the current state of all agents.

Entry types surfaced in UI: 💬 Update · ✅ Done · 📌 Task · ❓ Question · 🔀 Decision

Phase 4 & Future Capabilities

Higher autonomy, memory, artifacts — the path from managed agents to self-organizing teams
✦ Praxio
CEO Daily Ops Mode
Phase 4
You open Praxio, say "run," and the agents move. No constant prompting. Agents pick up the work queue in priority order, execute, surface results in Today, and flag decisions that need you. Rosalind coordinates sequencing — she knows who's free, what's blocked, and what needs a human call.

This is the self-driving organization. Today gives you the stream. Daily Ops Mode means the stream keeps moving even when you're heads-down on something else. Built on top of: memory ✓, Today view, live agents, work queues.
★ Phase 4 anchor New paradigm needs Today + live agents + work queues

Praxio Logo & Visual Identity
Phase 2
Praxio needs its own mark. "Praxio" comes from praxis — practice, action, execution. The logo should suggest forward motion and conversation, not management overhead. SVG-first so it works at any size in the UI. Needed before any external positioning begins.
